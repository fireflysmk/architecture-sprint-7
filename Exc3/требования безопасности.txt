==============================================
Требования к безопасности:
==============================================

1. шифрование данных:
- Все данные, передаваемые между системами PropDevelopment и партнерской платформой, должны шифроваться с использованием TLS 1.2/1.3
- Данные, хранящиеся в системах (например, биометрические данные), должны шифроваться на уровне базы данных

2. Аутентификация и авторизация:
- Использование OAuth 2.0 для аутентификации и авторизации между системами
- для доступа к API партнерской платформы должны использоваться токены доступа (access tokens) с ограниченным сроком действия
- Реализация ролевой модели доступа для ограничения прав пользователей

3. контроль доступа:
- ограничение доступа к API партнерской платформы только с доверенных IP-адресов
- использование межсетевого экрана для фильтрации входящего и исходящего трафика

4. Мониторинг и аудит:
- ведение журнала всех операций, связанных с использованием API партнерской платформы
- регулярный аудит доступа к API и анализ подозрительной активности

5. Протоколы аутентификации и авторизации:
- использование протокола OAuth 2.0 для аутентификации и авторизации
- токены доступа (access tokens) должны иметь ограниченный срок действия (например, 1 час)
- использование refresh tokens для обновления access tokens
- для аутентификации пользователей через мобильное приложение использовать OpenID Connect
- для аутентификации системного взаимодействия между PropDevelopment и партнерской платформой использовать API Keys

6. организация взаимодействия между системами:
- все запросы к партнерской платформе должны проходить через API Gateway, который обеспечивает: аутентификацию и авторизацию,  шифрование данных

7. логирование и мониторинг
- использование JSON для обмена данными между системами
- все данные должны быть валидированы перед передачей

8. обработка ошибок:
- реализация механизма обработки ошибок, включая: логирование ошибок, систему алертов, автоматическое восстановление после сбоев

9. резервирование и отказоустойчивость:
- обеспечение отказоустойчивости за счетиспользования нескольких экземпляров API Gateway
- реализация механизма повторной отправки запросов в случае сбоев
